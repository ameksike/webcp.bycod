La biblioteca denominada View se responsabiliza de incorporar al framework Bycod soporte para el renderizado de vistas a partir de plantillas, de una forma simple y elegante. Depende directamente de la biblioteca Notary en su versión 1.0, y requiere el intérprete del lenguaje PHP en su versión 5.2 o superior. Esta incorpora un conjunto de recursos en aras de facilitar el proceso de construcción de vistas, como es el caso de la generación automática de las etiquetas de tipo script o link, así como la inclusión de plantillas precompiladas. Para mayor comprensión obsérvese la definición de la interfaz que ofrece la mencionada biblioteca y que se muestra a continuación.

Interfaz de la biblioteca

            class View
            {
                public function render(\$view=":", \$data){}
                public function compile(\$view=":", \$data){}

                public function js(\$js="main", \$module){}
                public function css(\$css="main", \$module){}

                public function script(\$js="main", \$module){}
                public function link(\$css="main", \$module){ }

                public function url(\$name=false, \$mod){}
            }

Nótese como se ofrece dos formas distintas para la generación de las etiquetas script o link, las cuales permiten incorporar al documento HTML, fragmentos de código JavaScript y CSS respectivamente. De esta forma se deja a decisión del desarrollador qué función le resulta más cómoda utilizar, pues js y script se comportan de la misma forma, al igual que css y link.
